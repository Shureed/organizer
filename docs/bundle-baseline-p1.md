## Summary

The organizer SPA currently ships a single 527.65 KB (uncompressed) JavaScript chunk containing all routes, modals, views, and shared components. This baseline will be reduced through code-splitting in P1 tasks T2–T10, which will:

- Extract top-level modal mounts into a single `App.tsx` location
- Lazy-load the five main views (`TodayView`, `CalendarView`, `IssuesView`, `InboxView`, `RecentsView`)
- Lazy-load the `LoginPage` behind an auth check
- Lazy-load `AddTaskDialog` behind FAB interaction
- Lazy-load both detail modals (`TaskDetailModal`, `InboxDetailModal`) behind open-state gates
- Prefetch view chunks on hover/focus/touchstart to minimize perceived latency

The `dist/stats.html` file (generated by `rollup-plugin-visualizer`) provides a treemap visualization of the module graph for each build, enabling verification that chunks are properly split and no unintended modules are retained in the entry path.
